## How to Use OSV

The `osv` CLI allows you to securely manage secrets using your operating system's native credential store (macOS Keychain, Windows Credential Manager, Linux Secret Service / Keyring). 

### OS Prerequisites (Linux)

On Linux, `osv` relies on the Secret Service API (via `libsecret`). You must ensure it is installed on your system to interact with the keyring.

```bash
# Debian / Ubuntu
sudo apt-get update && sudo apt-get install libsecret-1-0

# Arch Linux
sudo pacman -S libsecret

# Rocky Linux / RHEL / Fedora
sudo dnf install libsecret
```
*(Note: A headless server might also require a keyring daemon like `gnome-keyring` and `dbus-run-session` to be configured for the secret service to run without a desktop environment).*

### Installation

Download the binary from the [releases page](https://github.com/frostyeti/osv/releases) or use a package manager:

```bash
# macOS/Linux (Homebrew)
brew install frostyeti/tap/osv
```

**Windows (Chocolatey)**
You can download and install the `.nupkg` directly from the release page:
```powershell
# Download the nuget package
curl -L -o osv.nupkg https://github.com/frostyeti/osv/releases/latest/download/osv.0.0.0-alpha.1.nupkg

# Install it locally
choco install osv -s .
```
*(Make sure to update the version number in the URL to the latest version)*

**Windows (Winget)**
You can download and install the Winget manifest zip directly from the release page:
```powershell
# Download the winget manifests zip
curl -L -o winget.zip https://github.com/frostyeti/osv/releases/latest/download/frostyeti.osv.winget-v0.0.0-alpha.1.zip

# Extract the manifests
tar -xf winget.zip

# Install using the local manifests
winget install -m .
```
*(Make sure to update the version number in the URL to the latest version)*

### Commands Reference

#### `osv set`
Store a secret in the vault. You can pass the value directly, via a file, stdin, or generate a random one.

```bash
# Value from command line
osv set --key api-token --value "super-secret"

# Generate a secure 32-character secret
osv set --key db-password --generate --size 32

# Value from a file
osv set --key cert --file ./cert.pem

# Value from stdin
echo "my-secret" | osv set --key stdin-secret --stdin
```

#### `osv get`
Retrieve one or more secrets from the vault. Format outputs as JSON, Dotenv, Bash exports, and more.

```bash
# Standard output
osv get api-token

# Output as dotenv file content
osv get api-token db-password --format dotenv

# Copy secret directly to clipboard (no console output)
osv get api-token --clip
```

#### `osv ls`
List all secret keys in the vault, or filter them using glob patterns.

```bash
# List all secrets
osv ls

# Filter by pattern
osv ls "api-*"
```

#### `osv rename`
Rename an existing secret key in the vault.

```bash
# Rename a secret key
osv rename api-token prod-api-token
```

#### `osv rm`
Permanently remove secrets from your vault.

```bash
# Remove a secret (prompts for confirmation)
osv rm prod-api-token

# Remove multiple secrets automatically without prompting
osv rm secret1 secret2 --yes
```

#### `osv config`
Manage `osv` configuration settings like default service names.

```bash
# Set a config value
osv config set service my-app-service

# Get a config value
osv config get service

# Remove a config value
osv config rm service
```

### Output Formats

The `get` command supports numerous output formats via `--format` allowing easy integration in your shell scripts or CI/CD pipelines:

- `text` (default): Raw text
- `json`: JSON object mapping keys to values
- `dotenv` / `env`: Key-value pairs for `.env` files
- `sh` / `bash` / `zsh`: Export statements for Unix shells
- `powershell` / `pwsh`: Environment variable assignments for PowerShell
- `github`: Registers masks and outputs GitHub Actions environment variable commands
- `azure-devops` / `ado`: Outputs Azure DevOps secret logging commands
